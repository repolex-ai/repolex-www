Classes (10)

ClassSubclass OfDescription
git:Actor — A person who performed an action in git (author, committer, or tagger). Identified by name and email.
git:Blob git:IndexObject A git blob object. Represents file contents. The same content always produces the same blob hash (content-addressed storage).
git:Commit git:Object A git commit object. Contains a snapshot (tree), parent commits, author/committer info, and a message.
git:IndexObject git:Object GitPython abstraction for objects that can appear in a tree index. Adds path, name, and mode information from the tree context. Parent class of Blob and Tree.
git:Object — Base class for all Git objects. All git objects are identified by their SHA-1 hash.
git:Reference — A git reference (ref). Includes branches, tags, HEAD, and remote-tracking branches. References are mutable pointers to commits.
git:Remote — A remote repository configuration (e.g., 'origin').
git:Repository — A git repository. Contains objects, references, and configuration.
git:Tag git:Object An annotated git tag. Contains tagger info, date, message, and points to a tagged object (usually a commit). Lightweight tags are just refs and don't have a Tag object.
git:Tree git:IndexObject A git tree object. Represents a directory - contains references to blobs (files) and other trees (subdirectories).
